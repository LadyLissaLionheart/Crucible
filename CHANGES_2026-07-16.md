## B1 — Layer model: global + page-scoped layers in layout.json  (09:45 MDT)
**Files:** `js/editor/layers.js`, `data/layout.json` (server-written), `js/editor/pagination.js`

Model persisted in `layout.json`:
```
layout.layers : flat array of { id, scope, page, name, visible, editHidden? }
  scope: 'global' (page=null, renders on EVERY page) | 'page' (renders only on its page)
  order in array = stacking order; LAST entry paints on top (highest z).
layout.entries[].layerId : points at a layer id.
```

`layers.js` exports the `Layers` module (IIFE, `const Layers = (()=>{...})()`).
Key members: `setup, sync, setTab, migrateLayout, ensureMigrated, currentPageNum,
getActiveLayerId, setActiveLayerId, selectLayerForEntry, isLayerHidden, addLayer,
deleteLayer, toggleVisible, renameLayer, reorderLayer, renderForPage,
computeCurrentPage, updateActiveLayerGroups`.

`migrateLayout(layout)`:
- ensures `layout.layers` is an array; each layer gets `id` (genId =
  `'layer-'+Math.random().toString(36).slice(2,9)`), `scope` (default 'global';
  global => page=null), `page` (number for page scope), `visible` (bool), `name`
  (default `'Layer '+(i+1)`).
- Restores `editHidden` from localStorage key `rulebook-layer-edit-hidden`
  (see B9); prunes ids no longer present.
- Ensures a Base layer exists (global) if array empty.
- Assigns any entry missing/with-stale `layerId` to the topmost layer
  (`layout.layers[last]`).
- Initialises module `activeLayerId` to the topmost layer if unset.

`layersForPage(page)` filters layers by `scope==='global' || l.page===page`.
`getActiveLayerId()` returns module `activeLayerId`, else topmost layer id.
`computeCurrentPage()` picks nearest non-appendix content page centre to the
viewport centre (appendix excluded). `contentPageCount()` = max entry page
(excludes appendix).

`pagination.js` `layOut()` → `layerGroupsFor(pageNum, pageEl)`: for each page,
build one `.layer-group` div per applicable layer (global + page-local),
`data-layer-id = l.id`, `style.zIndex = 2 + i` (above `.page-margin` z1), and
`[hidden]` when `l.visible===false` OR (edit mode AND `l.editHidden===true`).
Each card is appended into its layer's `.layer-group`. Groups are absolute
stacking contexts; per-entry `--z` still orders cards within a layer.

## B4 — Floating drag handle + live drag-and-drop reorder  (11:00 MDT)
**Files:** `js/editor/layers.js`, `css/layers.css`

Replaced native HTML5 DnD with a custom pointer-drag:
- Each row has a `.layer-grip` button (front) with a `grip-vertical` FA icon.
  `mousedown` (button 0 only — see B10) calls `startDrag(row, id, e)`.
- `startDrag` clones the row into a `.layer-floating` fixed-position element
  that follows the cursor (`e.clientX - dragOffset`, `e.clientY - dragOffset`),
  tilted upward via CSS `transform: rotate(-4deg)`. Source row gets `.dragging`
  (faded placeholder; `> * { visibility:hidden }`).
- `onMove` repositions the floating clone and calls `reorderLayerLive(...)` as
  soon as the cursor is over another row (NO midpoint delay, NO dashed
  insertion bar) — layers shuffle live.
- `onUp` removes the clone, clears `.dragging`, and calls `EditMode.setDirty()`.

Reorder math accounts for the panel showing layers REVERSED (top = highest z =
last in array). Helpers:
- `reorderVisual(srcId, targetId, after)`: builds the page's visual (top-first)
  order, removes src, inserts at target +/- 1, then `applyVisualOrder(visual)`.
- `applyVisualOrder(visualTopFirst)`: rebuilds `layout.layers` as
  `visual.reverse()` (+ any layers not in the page list) — preserves global vs
  page scope and other pages.
- `reorderLayerLive` = `reorderVisual` + `relayout()` (no dirty until drop).
- `reorderLayer` = `reorderVisual` + `setDirty` + `relayout()` (drop/legacy).
`relayout()` runs `StructureUI.repack()` (wrapped in try/catch) when
`EditMode.isActive()`, else `PageNumbers.paginate()`, then `renderForPage`.

CSS: `.layer-grip` styled (grab cursor); `.layer-row.layer-floating` fixed,
tilted, shadowed, `pointer-events:none`, z-index 1000; `.layer-row.dragging`
faded; rows have `transition: transform .12s ease` for live shuffle; NO
`.drop-before`/`.drop-after` dashed bar (removed).

## D0 — Floating entry UI no longer deselects the entry on click  (14:05)
**File:** `js/editor/structure.js` — function `onDeselectOutside(e)`
Symptom: clicking any entry UI option (Kind▾, title, Edit, Hide, z-up/down,
Delete, resize handles) made the UI vanish, because those controls were appended
to `.page` (outside the `.grid-card`) and `onDeselectOutside` cleared selection on
any `mousedown` whose target was not inside a `.grid-card`.
Fix: add an early-return guard before `clearSelection()`:
```js
function onDeselectOutside(e) {
  if (drag || resize || placement) return;
  if (e.button !== 0) return;
  const card = e.target.closest && e.target.closest('.grid-card');
  if (card) return;
  // Floating entry UI lives on .page, OUTSIDE the card, so it is never inside a
  // .grid-card. Clicking it must not deselect the entry (which would hide the UI).
  if (e.target.closest && e.target.closest('.entry-actions, .entry-edge-actions, .resize-handle, .entry-add-actions, .struct-btn')) return;
  clearSelection();
}
```

## D1 — Overflow frame moves INSIDE the card (no longer pops through entries)  (14:20)
**File:** `js/editor/structure.js` — `checkOverflow()` and `syncFixedUI()`
Symptom: the orange out-of-bounds frame was rendered `position: fixed; z-index:9998`
on `.page`, so it floated ABOVE everything (popped through entries stacked in
front of its own entry). It must sit with its entry content: above its own card
but below other entries with higher z.
Fixes:
- In `checkOverflow()`, APPEND the frame to the CARD (not `.page`):
  ```js
  card.appendChild(frame);
  ```
  (previously `(pageEl || card).appendChild(frame)`). The card is
  `position:absolute; z-index:var(--z,5)`, so the frame shares its stacking
  context and is ordered by `--z` among sibling entries. The frame uses
  CARD-LOCAL coordinates already computed by checkOverflow, so it must be
  `position:absolute` (see D2 CSS), NOT fixed.
- Keep the frame creation query as `card.querySelector('.overflow-frame[...]')`
  (both the early-return cleanup at top of checkOverflow and the main find).
- In `syncFixedUI()`: REMOVE the block that positioned the frame with
  `frame.style.top = (rect.top / zoom)` / `left` — that was for the old fixed
  positioning. The frame is now absolutely positioned inside the card via the
  card-local band coords set in checkOverflow, so syncFixedUI must NOT touch it.
- Do NOT add any inline opacity override on the frame. Because the frame is now a
  child of the card, it automatically inherits the card's faded `opacity:0.32`
  when the entry is hidden (`.grid-card.entry-hidden`), so the spill reads dimmed
  with no extra code.

## E1 — flushPendingEdits writes cached content on Save  (~15:50)
**File:** `js/editor/structure.js` — `flushPendingEdits()`

Extended `flushPendingEdits()` (called by `saveAll` AND `saveAndExit`) so it
also persists, for every entry still in the layout, its content from the
client-side `contentCache` (via `Renderer.getCachedContent(id)`), when there is
no newer pending edit for that entry.

```js
async function flushPendingEdits() {
  const layout = Renderer.getLayout();
  const entries = (layout && layout.entries) || [];
  const ids = new Set();
  for (const [id, html] of pendingEdits) {          // pending edits win
    ids.add(id);
    try { await API.saveEntry(id, html); }
    catch (err) { console.warn('flushPendingEdits: failed to save ' + id, err); }
  }
  for (const item of entries) {                      // re-persist cached content
    if (ids.has(item.id)) continue;
    const cached = Renderer.getCachedContent(item.id);
    if (cached === undefined) continue;
    try { await API.saveEntry(item.id, cached); }
    catch (err) { console.warn('flushPendingEdits: failed to restore ' + item.id, err); }
  }
  pendingEdits.clear();
}
```
Why: an undone delete restores the entry into the layout + contentCache, but its
server file was already gone (pre-E2). Writing the cache back recreated the file
with real content so the post-save reload showed correct content instead of the
auto-heal placeholder. (This becomes belt-and-suspenders once E2 lands, since
the file is never deleted at delete-time, but it is kept as a safety net.)

## G7 — MAIN FIX: entry UI -> position:fixed, moved to .page  (19:10)  [WIPED — RE-APPLY]
PROBLEM: entries on different layers occupy the same grid cells. A non-selected
layer's entry UI (actions bar, edge buttons, resize handles, overflow frame) was
rendered INSIDE the `.grid-card`, which lives inside a `.layer-group` that has an
inline `z-index` (`pagination.js` `g.style.zIndex = String(2 + i)`). That makes
each `.layer-group` its own stacking context, so a `position: fixed` child's
`z-index: 9999` was TRAPPED inside its own layer group and could never paint above
an entry on a higher layer.

SOLUTION: move the UI elements from the card to the `.page` element (sibling of
all `.layer-group`s) and tag them with `data-entry-id`, so they live in the page's
stacking context — above every layer group. `syncFixedUI()` then positions them
with `getBoundingClientRect()`, compensated for the `#zoom-container` zoom.
